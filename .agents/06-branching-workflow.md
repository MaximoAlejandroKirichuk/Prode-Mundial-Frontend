# Skill: Branching Workflow

## Use When
- Se pide crear una rama o naming.
- Se pregunta desde que rama partir.

## Goal
Mantener un flujo simple y consistente con el repo.

## Working Rules
- La base por defecto es `main` salvo pedido explicito.
- No crear, cambiar, mergear, rebasear, pull o push sin pedido explicito.
- No usar prefijo `codex/`.
- Formato sugerido: `modulo/accionConcreta`.
- Usar nombres cortos en ingles, sin terminos vagos.

## Naming Examples
- `auth/fixLoginRedirect`
- `home/addEmptyState`
- `universidades/improveListLoading`

## Output
- rama base recomendada
- nombre recomendado
- flujo de merge
- warnings si el pedido no encaja
